1. package.json - основа проекта

Объяснение: Это конфигурация npm проекта.
Определяет зависимости Angular и скрипты для запуска/сборки.

2. angular.json - конфигурация Angular

Объяснение: Конфигурация Angular CLI. Определяет как собирать проект,
куда складывать файлы и настройки для разработки/продакшена.

3. tsconfig.json - настройки TypeScript

Объяснение: Настройки TypeScript компилятора.
Определяет версию JavaScript,
строгие проверки типов и настройки для Angular.

4. src/index.html - главная HTML страница

Объяснение: Базовая HTML страница.
Тег <app-root> - это место куда Angular вставит наше приложение.

5. src/main.ts - точка входа

Объяснение: Запускает Angular приложение.
Это первая точка входа при загрузке.

6. src/styles.css - глобальные стили

Объяснение: Глобальные CSS стили применяются ко всему приложению.

7. src/environments/environment.ts - настройки для разработки

Объяснение: Переменные окружения для разработки.
apiUrl указывает на локальный бэкенд.

8. src/environments/environment.prod.ts - настройки для продакшена

Объяснение: Для продакшена используем пустой URL,
так как nginx будет проксировать запросы к бэкенду.

9. src/app/models/entry.ts - модель данных

Объяснение: TypeScript интерфейс точно повторяет структуру данных из бэкенда.
Это обеспечивает типобезопасность.

10. src/app/todo-list.service.ts - сервис для работы с API

Объяснение: Сервис общается с твоим бэкендом через HTTP запросы.
Использует environment.apiUrl для правильного URL.

11. src/app/app.module.ts - корневой модуль

Объяснение: Корневой модуль Angular.
Регистрирует компоненты и импортирует необходимые модули.

12. src/app/app.component.ts - главный компонент

Объяснение: Главный компонент управляет состоянием приложения,
обрабатывает пользовательские действия и взаимодействует с сервисом.

13. src/app/app.component.html - шаблон компонента

Объяснение: HTML шаблон с Angular директивами.
*ngFor для циклов, *ngIf для условного отображения,
[(ngModel)] для двусторонней привязки данных.

14. src/app/app.component.css - стили компонента

Объяснение: CSS стили только для этого компонента.
Используем современный CSS с Flexbox для адаптивного дизайна.

15. nginx.conf - конфигурация для Docker

Объяснение: Nginx слушает на порту 80,
раздает статические файлы Angular и
проксирует API запросы к бэкенд контейнеру.

16. Dockerfile - сборка фронтенда

Объяснение: Многоступенчатая сборка Docker.
Сначала собираем Angular,
потом настраиваем nginx для раздачи статических файлов.
